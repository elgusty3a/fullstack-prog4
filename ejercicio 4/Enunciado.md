# Ejercicio 4 — Diseño libre con Interfaces y Herencia

## Diseñar un pequeño sistema relacionado con métodos de pago.

Se espera que utilicen:

-   al menos una interface
-   una clase base
-   múltiples clases que extiendan de esa clase
-   clases que implementen la interfaz

Los atributos y métodos quedan a elección, pero deben tener sentido dentro del dominio elegido.

Ejemplos posibles de métodos de pago:

-   tarjeta de crédito
-   transferencia bancaria
-   billetera virtual

Al final del ejercicio se debe poder crear varias instancias de distintos métodos de pago y ejecutar una operación de pago.

El diseño de las clases y responsabilidades queda a criterio del estudiante.
